# 데이터·평가 프로토콜

이 디렉터리는 “정답 데이터셋”이 아니라, 서로 다른 시대의 불완전한 증거를 감사 가능한 범위로 바꾼 연구 장부입니다.

## 1. 분석 대상

- 남자 성인 11인제 축구 선수 경력
- 공식 1군 경기와 성인 A매치
- 감독 경력, 인기, 문화적 영향, 상업성은 점수 밖 서사
- 친선·전시·투어·리저브 경기는 공식 경쟁 기록과 분리
- 승부차기 득점은 일반 득점과 분리
- 진행 중 대회는 완료·검증 전까지 확정 점수에서 제외

증거 잠금일은 `candidates.json`과 `model.json`에 중복 기록하며 빌드가 이를 공개 페이지 상단에 표시합니다.

## 2. 후보 선정

후보군 자체가 결론을 만들지 않도록 시대와 역할을 먼저 층화합니다.

- 골키퍼
- 중앙 수비수·리베로·풀백
- 수비·중앙 미드필더
- 공격형 미드필더·윙어
- 센터포워드

각 층에서 공식 대회 개인상, 동시대 평가, 독립 전문가 패널, 검증 가능한 경기·시즌 통계 가운데 둘 이상의 독립 경로에서 역사적 최상위권인 선수를 포함합니다. Ballon d'Or는 1995년 전후 수상 자격이 달랐으므로 단독 후보 필터로 쓰지 않습니다. 후보 포함은 GOAT 판정이 아닙니다.

## 3. 다섯 핵심 축과 보조 업적

| 차원 | 기본 비중 | 질문 |
|---|---:|---|
| Peak 3 | 30% | 최고 3개 시즌의 동시대 상대 개인 기여가 얼마나 높았나 |
| Prime 5 | 20% | 최고 연속 5개 시즌에 그 수준을 얼마나 유지했나 |
| Career AUC | 25% | 엘리트 기준선을 넘은 시즌 가치를 얼마나 누적했나 |
| 클럽 맥락 | 10% | 리그·대륙대항전·역할 변화에도 영향력이 유지됐나 |
| 대표팀 맥락 | 10% | 국가 전력과 대회 접근성 차이를 감안해 경쟁 A매치에서 무엇을 더했나 |
| 업적 이력 | 5% | 개인상·우승이라는 불완전한 보조 증거가 얼마나 강한가 |

득점·도움·우승을 별도 차원으로 반복해서 넣지 않습니다. 이 사실들은 위 차원의 근거를 검증하는 입력이며, 상관된 증거의 이중 계산을 최소화합니다.

## 4. 범위 코딩

각 차원은 `[low, mode, high]`로 저장합니다.

- `mode`: 현재 증거에서 가장 타당한 중심 판단
- `low`: 자료 충돌·불리한 맥락 보정을 반영한 보수적 하한
- `high`: 누락 자료·역할 가치가 유리하게 해석될 때의 상한

세 값은 모두 0~100이고 `low <= mode <= high`여야 합니다. 100은 완전무결함이 아니라 해당 차원의 역사적 이론 상한입니다. 과거 자료가 적거나 제공자 정의가 충돌하면 점수를 자동으로 낮추지 않고 구간을 넓힙니다.

## 5. 불확실성·민감도

분석기는 각 차원을 삼각분포로 표집합니다. 가치 가중치는 균형·피크·지속성·대표팀·클럽·포지션 공정의 여섯 공개 시나리오를 같은 확률로 고른 뒤 그 주변의 제한된 Dirichlet 분포에서 표집합니다.

40,000회 반복해 다음을 산출합니다.

- 기본 순위와 구조적 하한·상한
- 1위 도달률과 Top 3 도달률
- 기본 Top 3 간 쌍대 우위 비율
- 시나리오별 순위와 최대 순위 변동
- 모든 차원에서 다른 후보에게 완전히 지배되지 않는 Pareto frontier

1위 도달률은 실제 세계의 확률이 아닙니다. 이 저장소가 허용한 입력 범위와 가치 판단 안에서의 강건성입니다.

## 6. 출처 계층

1. FIFA·UEFA·CONMEBOL·IFAB·France Football의 공식 대회·규칙·수상 자료
2. 역할 가치와 모델 설계를 위한 동료평가 연구·저자 공개 논문
3. 공식 자료의 역사 공백을 보완하는 RSSSF 같은 전문 2차 아카이브
4. 편향을 감사한 뒤 특정 완전 대회 사례에만 쓰는 공개 이벤트 데이터

출처가 충돌하면 숫자를 조용히 하나로 합치지 않습니다. 범위를 넓히고 `evidence_note`, `case_for`, `case_against`에 충돌 이유를 남깁니다.

## 7. 갱신 승인 규칙

- 완료된 대회만 확정 스냅샷으로 승격
- 활동 중 선수의 새 기록은 동일 범위 정의로 검증
- 새 지표가 과거 후보에게 없으면 현대 심층 분석으로 분리
- 점수 상향에는 반대 근거 검토, 하향에는 찬성 근거 검토를 동시에 수행
- 후보·차원·출처 ID와 가중치 합계 테스트를 반드시 통과
- 모델 버전은 루브릭·가중치·표집 전략이 바뀔 때 증가

모든 공개 표와 차트는 `scripts/analyze.py`와 `scripts/build_report.py`에서 다시 생성됩니다.
